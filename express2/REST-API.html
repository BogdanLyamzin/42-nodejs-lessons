Старый вариант: / getAllProducts / getOne / addProduct / updateOneProduct 1. 1. Ендпоинты - это существительные во множественном числе. 2. Глаголы в адресе недопустимы 3. Тип CRUD операции определяется методом HTTP-запроса. GET /products - верни все товары GET /products/:id - верни товар с этим id POST /products - добавить один товар PUT /products/:id - возвращает все товары POST: добавляет новый товар /id PUT: обновляет полностью DELETE PATCH 4. С одной сущностью работают по /id 5. версию указывайте в начале адреса: /api/v1/products [{}] /api/v2/products { page: 1, totalPage: 20, results: [] } Ответы: 1. Код ответа определяет реакцию фронтенда, или "Код ответа показывает, как прошла обработка операции на сервере" 200-299 - обработка прошла по плану 300-399 - страница/роут перемещены на другой адреса 400-499 - ошибка запроса (ошибка со стороны фронтенда) 500+ - ошибка сервера 200 - все ок 201 - Элемент создан (на POST запрос) 204 - No content (успешный DELETE-запрос) 301 - содержимое было перемещено на постоянной основе 302 - временный переезд 400 - Bad Request - чего-то в запросе не хватило (токена, полей и т.д.) 401 - Не авторизирован 403 - Не хватает прав 404 - не существует 500+ - ошибки сервера Ответ на POST запрос - добавленный объект + id
